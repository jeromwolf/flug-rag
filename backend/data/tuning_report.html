적용 튜닝 기법

Phase 2에서 적용한 9가지 RAG 품질 개선 기법

1
System Prompt Optimization
법률/기술/일반 도메인별 특화 시스템 프롬프트 적용. 7B 모델용 간결성 지시 포함. 한국어 응답 품질 최적화.
2
Few-Shot Examples
factual, inference, negative 패턴을 커버하는 20개 큐레이션된 예시. 답변 형식과 길이 가이드 역할.
3
Category-Specific Weights
factual: ROUGE 가중 (정확한 용어 매칭), inference: semantic 가중 (의미적 이해도), negative: semantic 지배적 (부정 판단 정확도).
4
Length Penalty
기대 길이 대비 2.5배 초과 답변에 점진적 감점 적용. 과잉 답변으로 인한 ROUGE 저하 방지.
5
Negation-Aware Scoring
골든 답변이 "없습니다"인데 실제 답변이 "있습니다"인 경우 0.3x 패널티. 14개 한국어 부정 표현 마커 활용.
6
Selective Multi-Query
multi_hop 질문 자동 탐지 (키워드 매칭), 해당 질문에만 multi-query 활성화 (top_k=30). 일반 질문 대비 3배 컨텍스트.
7
Negative Question Prompt
"~있나요?" 패턴 질문에 5가지 엄격 가이드 적용: 직접 규정 확인, 간접 참조 배제, 불확실시 부정 응답.
8
HyDE (Query Expansion)
Hypothetical Document Embedding: 질문에서 가상 문서를 생성하여 임베딩 후 검색. Recall 향상에 기여.
9
FlashRank Reranker
ms-marco-MultiBERT-L-12 모델로 검색 결과 재순위화. 초기 검색의 노이즈를 필터링하여 정밀도 향상.
