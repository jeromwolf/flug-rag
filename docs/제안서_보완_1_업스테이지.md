### 위치 1: II. 제안시스템 구축 > 2.2 핵심 기술 > 문서 수집 및 전처리

**현재 제안서 내용 (line 135):**
> "공사의 주요 문서 유형인 사규/지침(HWP), ISO 표준(PDF), 법령정보, 기술 매뉴얼(DOCX)을 Upstage Document Parse를 통해 파싱하며..."

**보완 제안:**

현재 한 줄로 간략히 언급되어 있으므로, 아래 내용을 추가하여 업스테이지 기술 역량을 구체적으로 기술:

> **Upstage Document Parse 기술 상세**
>
> 본 플랫폼은 Upstage Document Parse를 핵심 문서 파싱 엔진으로 채택하였다. Upstage Document Parse는 글로벌 문서 인식 벤치마크에서 Google Document AI, Microsoft Azure AI Document Intelligence 대비 5%+ 우위(TEDS 93.48)를 기록한 업계 최고 수준의 문서 파싱 솔루션이다.
>
> **주요 기술 역량:**
> - **HWP/HWPX 네이티브 파싱**: PDF 변환 없이 HWP 파일 직접 파싱 (2025.4월 `document-parse-250404` 버전부터 지원). 공사의 핵심 문서인 사규/지침 676건의 HWP 파일을 원본 구조 그대로 처리 가능
> - **다단 표 구조 복원**: 병합 셀, 숨겨진 격자선, 다중 페이지 표 자동 병합 지원. 배관 스펙 시트의 복잡한 다단 표 구조를 HTML `<table>`로 정확히 변환
> - **12종 요소 분류**: table, figure, chart, heading1, header, footer, caption, paragraph, equation, list, index, footnote — 문서 구조를 자동으로 분류하여 후처리 파이프라인에서 요소별 차등 처리 가능
> - **좌표 정보 제공**: 정규화된 4점 바운딩 박스(0~1 범위)로 원본 문서 내 위치 매핑, OCR 검수 인터페이스에서 원본-텍스트 대조에 활용
> - **차트/그래프 인식**: 차트를 구조화 데이터 + 자연어 설명으로 변환하여 RAG 검색 대상에 포함

### 위치 2: II. 제안시스템 구축 > 2.2 핵심 기술 > OCR 후처리 및 데이터 정제 (표 II-6 앞)

**현재 제안서 내용 (line 160):**
> "본 플랫폼은 Upstage Document Parse 기반 OCR 이후, 6단계 자동 정제 파이프라인을 거쳐..."

**보완 제안 — 업스테이지 제공 범위 vs 자체 구현 범위 명확화:**

[표 II-6] 뒤에 아래 표를 추가:

| 단계 | 처리 내용 | 업스테이지 커버 | 자체 구현 | 비고 |
|------|----------|:---:|:---:|------|
| 1. 구조 검증 | 표/리스트/제목 구조 정합성 확인 | **부분** | **O** | 12종 element 태깅 제공, 검증 로직은 자체 구현 |
| 2. 오인식 보정 | 가스 설비 전문용어 사전 기반 교정 | **X** | **O** | 커스텀 사전 미지원 → 전량 자체개발 (KGS/KOSHA 코드 사전) |
| 3. 노이즈 제거 | 머리글/꼬리글/워터마크 제거 | **대부분** | **일부** | header/footer 분류 필터 가능, 워터마크는 자체 처리 |
| 4. 텍스트 정규화 | 특수문자, 인코딩, 공백 통일 | **부분** | **O** | UTF-8 출력 기본, ㎥→m³ 등 단위 정규화는 자체 구현 |
| 5. 메타데이터 추출 | 문서명, 작성일, 버전, 부서 | **X** | **O** | regex/NER 기반 자체 추출 |
| 6. 품질 점수 산정 | 글자 수, 구조, 오인식률 종합 | **X** | **O** | confidence API 미노출, 자체 산정 알고리즘 |

> **업스테이지 네이티브 커버**: 3단계(노이즈 제거) 대부분, 1단계(구조 태깅)·4단계(인코딩) 부분 지원
>
> **전량 자체 개발**: 2단계(전문용어 사전), 5단계(메타데이터), 6단계(품질 점수)

### 위치 3: II. 제안시스템 구축 > 시스템 아키텍처 (비교표 부근, line 130)

**현재 제안서 내용:** 기존 방식과의 비교표에 "Tesseract, Naver Clova OCR"이 기존 기술로 언급됨

**보완 제안 — 업스테이지 선택 근거 표 추가:**

| 비교 항목 | Tesseract | Naver Clova OCR | **Upstage Document Parse** |
|-----------|-----------|-----------------|---------------------------|
| HWP 지원 | X (PDF 변환 필요) | X | **O** (네이티브) |
| 표 구조 인식 | 기본 수준 | 보통 | **TEDS 93.48** (업계 최고) |
| 온프레미스 배포 | O (오픈소스) | X (클라우드 전용) | **O** (Docker 이미지) |
| 폐쇄망 지원 | O | X | **O** (`docker save/load`) |
| 요소 분류 | X | 제한적 | **12종 자동 분류** |
| 좌표 정보 | O (기본) | O | **O** (정규화 4점) |
| 라이선스 | Apache 2.0 | 상용 (클라우드) | 상용 (온프렘 라이선스) |

> **선정 사유**: 공사의 핵심 문서 형식인 HWP 네이티브 파싱 지원, 폐쇄망 온프레미스 배포 가능, 표 구조 인식 업계 최고 수준의 세 가지 요건을 모두 충족하는 유일한 솔루션
