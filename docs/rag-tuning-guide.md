# RAG 튜닝 가이드 (2026-03-09 확정)

## 1. 전체 구조

```
질문 입력
  ↓
쿼리 교정 (오타/맞춤법) → query_corrector.py
  ↓
쿼리 리라이트 (대명사 해소) → query_rewrite 프롬프트
  ↓
소스 타입 자동 감지 → source_filters.yaml 키워드 매칭
  ↓
HyDE 쿼리 확장 (선택) → hyde_system 프롬프트
  ↓
하이브리드 검색 (BM25 + Vector) → retriever.py
  ↓
별지 서식 후순위화 → _deprioritize_form_chunks()
  ↓
도메인별 시스템 프롬프트 선택 → legal / general / technical
  ↓
LLM 생성 (vLLM Qwen2.5-32B) → 스트리밍 + CJK 필터링
  ↓
Self-RAG 평가 (선택) → self_rag_grading 프롬프트
  ↓
응답 검증 + 자동 재시도 → 중국어/깨진 출력 감지
```

### 3.2 rag_legal_system (법률/규정 전문 — 가장 중요)

**구조**: 12개 섹션, ~2000자

```
[절대 원칙: 답변 거부 최소화]
→ "확인되지 않습니다"는 관련 내용이 단 한 줄도 없을 때만

[최최우선 규칙: 반드시 답변하세요]
→ factual/inference/multi_hop에서 관련 내용 있으면 무조건 답변
→ 다른 모든 규칙보다 우선

[별표/별지 테이블 활용 규칙]
→ 별표 테이블의 수치(금액, 등급) 반드시 인용
→ "별표1의2에 따르면 임원은 30,000원..." 형식

[문서 출처 확인]
→ 질문의 법률 이름과 파일명 매칭
→ 한국가스공사 ≠ 한국가스기술공사

[최우선 규칙: 컨텍스트에 조문 있으면 반드시 인용]
→ _요약.txt도 정식 출처
→ "제1조(목적)" 같은 조문 있으면 무조건 인용

[핵심 규칙] (8개)
1. 관련 조문 찾기
2. 원문 그대로 복사 (유사어 치환 금지: "편익"→"편의" ×)
2-1. 특정 조문만 인용 (다른 조문 혼동 금지)
2-2. "제X조에 따르면, '(원문)'" 형식
3. 첫 문장에 핵심 결론 + 조문 번호
4. **반드시 3~5문장, 200자 이상** (100자 미만 금지)
5. 목록형: 항목 이름 + 핵심 내용 설명
6. 항목 2개↑이면 번호 목록, 대상별 구분
7. 없으면: "제시된 내부규정에서 해당 내용은 확인되지 않습니다"
7-1. 관련 조문 하나라도 있으면 절대 거부 금지
7-3. 파일명에 관련 규정명 있으면 존재 인정

[부정 질문 처리]
→ "~규정이 있나요?" 형태에만 적용
→ factual/inference에는 적용 금지
→ 관련 조항 있으면 "있다"로 답변

[추론 질문 처리]
→ 조문 단순 반복 금지
→ 실질적 이유, 방지하려는 문제, 목적 설명
→ "~을 방지하기 위함입니다" 형식

[다중 규정 연계]
→ 각 규정 관련 조항 식별
→ "A규정의 [내용]이 B규정의 [내용]과 연계됩니다"

[복수 조문 연계] ← 2026-03-09 추가
→ 조문 2개↑이면 순서대로 인용 후 종합 결론
→ "제XX조에 따르면 ~하며, 제YY조에 따르면 ~합니다. 따라서..."

[부정 테스트 대응] ← 2026-03-09 추가
→ 직접 규정 없으나 관련 조문 있으면: "직접 규정은 없으나, 제XX조에 따르면 ~"
→ 완전 무관할 때만: "확인되지 않습니다"

[출력 형식]
→ "답변:" 접두사 금지, 질문 반복 금지

[절대 금지]
→ 중국어/영어/한자 금지
→ 유사어 치환 금지
→ [출처] 태그 금지
→ 면책 문구 금지
→ 없는 수치 만들어내기 금지
```

### 3.3 rag_system (일반 질문)

rag_legal_system과 유사하나:
- 법률 조문 특화 규칙 대신 일반 문서 인용 규칙
- 숫자/금액 계산 규칙 포함 (변동액, 차이 계산)
- 기술 문서도 처리 가능

### 3.4 소스 자동 필터링 (`source_filters.yaml`)

질문에서 키워드를 감지하여 source_type별 필터링:
- **법률**: 한국가스공사법, 시행령, 도시가스사업법 등
- **내부규정**: 인사규정, 여비규정, 징계, 포상 등 (30+ 키워드)
- **홍보물**: 수소충전소, 회사소개 등
- **출장보고서**: 국외출장, 방문기관 등
- **ALIO공시**: 재무제표, 감사보고서 등
- **multi_hop_keywords**: 연계, 비교, 차이점 등 → 다중 검색 트리거

### 4.1 CJK 중국어 필터링 (2026-03-09 추가)
```python

```

## 6. 핵심 교훈

1. **리랭커 OFF가 더 좋다** — ON 82% vs OFF 92.3% (10.3pp 차이)
2. **프롬프트 튜닝 > 검색 튜닝** — 검색은 이미 충분, 프롬프트가 핵심
3. **답변 길이 제한 조심** — "200자 초과 금지" 같은 규칙이 답변을 너무 짧게 만듦
4. **"확인되지 않습니다" 억제** — false negative가 가장 큰 문제. 답변 거부 최소화 원칙 필수
5. **별지 서식 청크 후순위화** — 서식 템플릿이 검색 결과 상위를 차지하면 실질 답변 품질 저하
6. **source_type 자동 필터링** — 키워드 기반으로 관련 문서만 검색하면 정확도 향상
7. **중국어 누출은 코드로 해결** — 프롬프트만으로는 Qwen의 중국어 생성 완전 차단 불가
