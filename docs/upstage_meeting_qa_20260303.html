A. 라이선스 & 비용
# 질문 배경 사전 조사 미팅 확인
A1 온프레미스 라이선스 모델은?
영구? 연간? 문서 단위?		 제안서에 "온프레미스 라이선스 구매" + "기술 지원 계약 별도 체결"로 기재. 정확한 가격 모델 필요 AWS SageMaker Marketplace를 통한 온프렘 배포 확인됨. 라이선스 모델 세부는 영업 확인 필요
A2 라이선스 비용 견적
GPU 1~2대, 폐쇄망, 월 500~1000건		 총 사업비 429,000천원 중 OCR 비용 미확정 (제안서에 "이부일 대표님 체크 필요"로 표기) Cloud API: $0.01/페이지. 온프렘은 별도 견적 필요
A3 Cloud API 가격
페이지당 단가? 볼륨 할인?		 POC/개발 단계에서 Cloud API 사용 중. 운영 전환 전까지 비용 예측 필요 $0.01/페이지 공식 단가. 볼륨 할인 여부는 영업 확인
A4 기술 지원 계약 범위와 비용
SLA, 응답 시간, 모델 업데이트		 제안서에 별도 체결로 기재했으나 구체적 조건 미확정 미팅 확인
B. 온프레미스 배포
# 질문 배경 사전 조사 미팅 확인
B1 최소 하드웨어 스펙
GPU 필수? CPU-only? VRAM?		 발주처 서버: L40S 48GB x4, CPU 16Core x2, RAM 256GB GPU/CPU 양쪽 Docker 이미지 제공 확인. 정확한 VRAM 요구량은 미팅 확인
B2 폐쇄망 Docker 이미지 반입
오프라인/에어갭 환경 지원?		 발주처가 폐쇄망(인터넷 차단). docker save/load 가능 여부 확인 AWS SageMaker 기반 온프렘 배포 확인됨. 순수 에어갭 가능 여부는 미팅 확인
B3 모델 버전 업데이트 주기
교체 절차? API 하위 호환성?		 현재 document-parse-240910 사용 중 최신: 250508. 릴리스: 240910→250404→250508.
HWP 모듈은 251217.1 이상 필요 (이전 버전 2025.12.31 라이선스 만료)
B4 동시 처리 성능
동시 요청 수, 페이지/분 처리량		 배치 인제스트 시 5건 병렬 처리. 초기 데이터 로딩 소요 시간 예측 필요 동기 API 최대 100페이지/요청, 비동기 API 최대 1,000페이지/요청. GPU별 벤치마크는 미팅 확인
C. 기술 기능
# 질문 배경 사전 조사 미팅 확인
C1 HWP 파일 직접 지원 여부
직접 파싱? PDF 변환 필요?		 발주처 핵심 문서 676건이 HWP. 현재 pyhwp로 텍스트 추출 중이나 표 구조 손실 지원됨 2025.4월 250404부터 HWP/HWPX 네이티브 파싱. PDF 변환 불필요. 표 구조를 HTML <table>로 반환. 최대 50MB.
주의 온프렘은 모델 251217.1 이상 필요 (HWP 라이선스 만료)
C2 표 구조 인식 정확도
다단 표, 병합 셀, 다중 페이지 표		 제안서에 "배관 스펙 시트의 다단 표 구조 복원" 약속 우수 병합 셀, 다단 표, 숨겨진 격자선, 다중 페이지 표 자동 병합. TEDS 93.48 (Google/MS 대비 5%+ 우위). 차트/그래프는 구조화 데이터 변환 가능
C3 응답 포맷
마크다운/HTML? 좌표?		 coordinates=true 옵션 사용 중. 반환 포맷 스펙 확인 JSON 반환: content.html/markdown/text (문서 전체) + elements[] 배열 (요소별).
12개 카테고리: table, figure, chart, heading1, header, footer, caption, paragraph, equation, list, index, footnote.
좌표: 정규화된 4점 바운딩 박스 (0~1 범위)
C4 confidence 점수 제공 여부
페이지별? 문서별?		 제안서에 "품질 점수 임계값 0.85" 기재. confidence 0.0 반환 가능성 API 미노출 내부적으로 워드 레벨 confidence 존재하나 표준 API JSON에 confidence 필드 없음. 페이지별/문서별 점수도 미제공.
→ 품질 점수는 자체 구현 필요
향후 노출 계획 확인 필수
C5 한국어 전문용어 커스터마이징
KGS, KOSHA 코드 사전 등록?		 제안서 6단계 중 2단계 "가스 설비 전문용어 사전 기반 교정"의 실현 가능성 미지원 커스텀 사전/어휘 등록 기능 없음. 도메인 용어 교정은 자체 후처리 파이프라인으로 구현 필요. 엔터프라이즈 커스텀 파인튜닝은 별도 협의
D. 6단계 OCR 후처리 파이프라인

제안서에 약속한 6단계에 대해 업스테이지 제공 범위 vs 자체 구현 범위를 사전 조사 결과로 정리합니다.

단계 처리 내용 제안서 예시 업스테이지 자체구현 사전 조사 결과 미팅 확인
1 구조 검증
표/리스트/제목 정합성		 배관 스펙 시트의 다단 표 구조 복원 부분 O 12개 element category 태깅 제공. 구조 검증 로직은 자체 구현 필요
2 오인식 보정
전문용어 사전 교정		 'KGS FP551' → 'KGS EP551' X O 전량 자체개발. 커스텀 사전 미지원. 가스 설비 전문용어 사전 + 매칭 로직 필요
3 노이즈 제거
머리글/꼬리글/워터마크		 '대외비' 워터마크, 페이지 번호 제거 대부분 일부 header/footer 별도 카테고리로 필터 가능. 워터마크 제거는 자체 구현
4 텍스트 정규화
특수문자/인코딩/공백		 ㎥→m³, ㎏/㎠→kg/cm² 부분 O UTF-8 출력 기본 제공. 단위 정규화, 특수문자 통일은 자체 구현
5 메타데이터 추출
문서명/작성일/버전/부서		 사규 개정 이력 자동 식별 X O 구조화된 메타데이터 추출 미지원. header/footer에서 regex/NER로 자체 추출
6 품질 점수 산정
글자 수/구조/오인식률		 임계값 미달 시 수동 검수 플래그 X O confidence 미노출. 글자 수/구조 완결성/한국어 비율 기반 자체 산정

사전 조사 요약

  • 네이티브 커버 3단계(노이즈 제거) 대부분
  • 부분 지원 1단계(구조 태깅), 4단계(인코딩)
  • 전량 자체 개발 2단계(전문용어), 5단계(메타데이터), 6단계(품질 점수)
⚠️ 핵심 확인 질문: 6단계 중 업스테이지가 추가 제공 가능한 기능이 있는지? 특히 confidence 점수 노출 계획과 커스텀 사전 지원 로드맵.
E. POC/데모 지원
# 질문 배경 사전 조사 미팅 확인
E1 트라이얼 API 키/크레딧
POC용 무료/할인?		 3/5 발주처 원본 데이터 수령 후 즉시 인제스트 필요. 수천 페이지 예상 공식 단가 $0.01/페이지. 수천 페이지 = $수십 수준. 트라이얼 크레딧 유무 미팅 확인
E2 샘플 문서 사전 테스트
HWP/PDF 5~10건 파싱 품질		 제안서에 "문서 파싱 품질 사전 검증" 약속 Cloud API로 즉시 테스트 가능. HWP 5~10건 사전 파싱하여 품질 확인 제안
E3 기술 지원 채널
Slack/이메일/전담 담당자?		 개발 기간 4개월. 빠른 이슈 해결 필요 미팅 확인
E4 레퍼런스/사례
공공기관 폐쇄망 배포, HWP 대량 처리		 발주처 설득 자료 및 리스크 판단용 미팅 확인
미팅에서 꼭 받아올 것 (체크리스트)
  • 온프레미스 라이선스 견적서 (제안서 비용 항목에 반영)
  • Cloud API 단가표 + 볼륨 할인 (POC 비용 산정)
  • HWP 온프렘 모델 버전 확인 (251217.1 이상인지) — 라이선스 만료 이슈
  • 폐쇄망 Docker 이미지 반입 절차 문서 (에어갭 docker save/load 가능 여부)
  • confidence 점수 향후 API 노출 계획 확인
  • 커스텀 사전/파인튜닝 지원 가능 여부 (가스 설비 전문용어)
  • 트라이얼 키 또는 크레딧 (3/5 데이터 인제스트용)
  • 6단계 파이프라인 중 추가 제공 가능한 기능 확인
  • 공공기관 폐쇄망 배포 레퍼런스/사례
  • GPU별 성능 벤치마크 (L40S 48GB 기준 페이지/분 처리량)
