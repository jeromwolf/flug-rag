# 발표자료 vs 실제 구현 검증 결과

> 검증일: 2026-03-04
> 대상: 한국가스기술공사 생성형 AI 플랫폼 제안 발표자료 (약 20장)
> 검증 방법: 발표자료의 모든 기술 주장을 코드베이스 대조 확인

---

## 1. 전체 요약

| 구분 | 비율 | 설명 |
|------|------|------|
| 완전 구현 | ~75% | 코드 존재, 작동 확인 |
| 코드 존재/활성화 필요 | ~15% | 구현되어 있으나 설정 변경 또는 연결 필요 |
| 신규 구현 필요 | ~10% | 코드 미존재, 개발 필요 |

## 2. 슬라이드별 검증 결과

### 2.1 사업 목표 및 범위 (3가지 축)

#### 축 1: 지능형 검색 체계 — 모두 구현 완료

| 항목 | 상태 | 근거 |
|------|------|------|
| RAG 기반 정확도 90%+ | ✅ 구현 | 벤치마크 95.8% (115/120) |
| 하이브리드 검색 | ✅ 구현 | `retriever.py` — 벡터(0.7)+BM25(0.3)+CrossEncoder 리랭킹 |
| 출처 명시형 응답 | ✅ 구현 | SSE source 이벤트 + confidence 점수 + 안전성 경고 |

#### 축 2: 업무 생산성 혁신 — 모두 구현 완료

| 항목 | 상태 | 근거 |
|------|------|------|
| 문서 요약/생성 | ✅ 구현 | `summarizer_tool.py` — 추출식+생성식 요약 |
| 보고서 초안 작성 | ✅ 구현 | `report_draft_tool.py` — 4개 보고서 유형, RAG 기반 |
| AI Agent 업무 자동화 | ✅ 구현 | MCP 13개 도구 + 체인 실행 + WebSocket 모니터링 |

#### 축 3: 지식 자산화 — 일부 갭

| 항목 | 상태 | 비고 |
|------|------|------|
| 암묵지→명시지 전환 | ✅ 구현 | `golden_data.py` — 전문가 피드백 → Q&A 큐레이션 |
| 도메인 특화 학습 | ❌ 미구현 | 프롬프트 엔지니어링만 적용, 실제 파인튜닝 없음 |
| 문서 버전 이력 관리 | ⚠️ 부분 | 프롬프트 버저닝 있음, 문서별 이력 조회 API 없음 |

### 2.2 추진 전략 및 핵심 접근법 (4개 영역)

#### 폐쇄형 On-Premise

| 항목 | 상태 | 비고 |
|------|------|------|
| 외부 API 의존 없는 완전 폐쇄형 | ✅ 구현 | Ollama/vLLM 로컬, 임베딩 로컬(bge-m3), Milvus Lite |
| 데이터 주권 확보, 보안 최우선 | ✅ 구현 | JWT/RBAC, PII 탐지, 가드레일 |
| H200 NVL x4 GPU 클러스터 | — | 인프라 영역 (코드 무관) |

#### 듀얼 LLM 전략

| 항목 | 상태 | 비고 |
|------|------|------|
| Qwen2.5-72B (고성능 메인) | ⚠️ 불일치 | 코드상 메인=32B-AWQ (72B는 운영 목표 사양) |
| Qwen2.5-32B (경량 보조) | ⚠️ 역전 | 32B가 메인, 14B가 보조로 설정됨 |
| Router 자동 분배 | ✅ 구현 | `router.py` — QueryRouter + ModelTier(MAIN/LIGHT) |
| 72B 장애 시 32B 자동 인수 | ❌ 미구현 | 페일오버 로직 없음 |
| 처리량 2.5배 | ❌ 미검증 | 벤치마크 데이터 없음 |

#### 4중 환각방지 체계

| 차수 | 항목 | 상태 | 비고 |
|------|------|------|------|
| 1차 | 출처 기반 생성 | ✅ 구현 | 컨텍스트 프롬프트 + sources 반환 |
| 2차 | Re-ranking 검증 | ✅ 구현 | bge-reranker-v2-m3 CrossEncoder |
| 3차 | Self-RAG 반성 | ⚠️ 비활성 | `self_rag.py` 존재, `SELF_RAG_ENABLED=false` 기본값, 스트리밍에 미통합 |
| 4차 | 가드레일 최종 필터 | ⚠️ 부분 | 규칙 기반만 구현, "LLM 기반 이중 필터" 주장이나 LLM 필터 미구현 |

#### MCP 기반 AI Agent

| 항목 | 상태 | 비고 |
|------|------|------|
| Router→Planner→Executor | ⚠️ 부분 | Router, Planner 있음. Executor 독립 클래스 없음 |
| 6개 업무 시나리오 자동화 | ⚠️ 4/6 | 설비 점검이력 조회(부분), 경영평가 자료 수집(미구현) |
| Tool Registry 기반 확장 | ✅ 구현 | 13개 도구 등록 + 스키마 + 커스텀 도구 빌더 |

### 2.4 4종 청킹 전략 (자동 선택) — 상세

| 전략 | 구현 파일 | 적용 대상 | 상태 |
|------|---------|---------|------|
| ① 시맨틱 | `EmbeddingSemanticChunker` | 기술문서·ISO | ✅ 구현 (bge-m3 코사인 유사도 경계 탐지) |
| ② 고정크기 | `RecursiveChunker` | 행정문서 | ✅ 구현 (한국어 인식 분리자 계층) |
| ③ 표구조보존 | `TableChunker` | 설비 스펙시트 | ✅ 구현 (행별 처리 + 헤더 보존) |
| ④ 계층적 | `HierarchicalChunker` | 법규·규정 | ✅ 구현 (제N조 부모-자식 구조) |
| 자동선택 | `AdaptiveChunker` | 문서 유형 자동 | ✅ 구현 but 기본 비활성 (`chunk_strategy="recursive"`) |
| DP 연동 | — | Upstage 요소 유형 기반 | ❌ 미구현 (요소 추출하나 청킹 라우팅 미연결) |

## 3. 발표자료에 미포함된 구현 완료 기능 (추가 추천)

### 공공기관 컴플라이언스 (제안서 추가 강력 추천)

| 기능 | 구현 파일 | 차별화 포인트 |
|------|---------|-------------|
| **한국형 PII 탐지** | `pipeline/pii_detector.py` | 주민번호·전화번호·계좌번호 등 7종 자동 마스킹. 개인정보보호법 필수 요건 |
| **AI 윤리서약** | `auth/ethics.py` | 버전 관리 + IP/시간 감사 로그. AI 윤리 가이드라인 대응 |
| **접근요청 워크플로** | `auth/access_request.py` | 역할 변경 신청→관리자 승인. 내부통제 감사 대응 |
| **남용탐지** | `monitoring/abuse_detector.py` | IP 블랙리스트 + 레이트리밋(30req/min) + 이벤트 영속화 |

### RAG 고급 기법 (기술 차별화 강화)

| 기능 | 구현 파일 | 효과 |
|------|---------|------|

| **HyDE 쿼리 확장** | `rag/query_expander.py` | 가설 문서 생성 → 임베딩 → 검색 정확도 향상 |
| **Agentic RAG** | `rag/agentic.py` | 질문 유형별 전략 자동 선택 (일반/멀티쿼리/다이렉트/딥서치) |
