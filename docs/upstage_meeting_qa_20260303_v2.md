## D. 6단계 OCR 후처리 파이프라인

제안서에 아래 6단계를 약속했으며, 각 단계별로 **업스테이지 제공 범위 vs 자체 구현 범위**를 사전 조사 결과로 정리합니다.

| 단계 | 처리 내용 | 제안서 예시 | 업스테이지 제공 | 자체 구현 | 사전 조사 결과 | 미팅 확인 |
|------|----------|-----------|:-:|:-:|------|------|
| 1. 구조 검증 | 표/리스트/제목 구조 정합성 확인, 파싱 오류 자동 보정 | 배관 스펙 시트의 다단 표 구조 복원 | **부분** | **O** | 12개 element category 태깅 제공 (table, heading1, list 등). 구조 검증 로직 자체 구현 필요 | |
| 2. 오인식 보정 | 가스 설비 전문용어 사전 기반 교정 | 'KGS FP551' → 'KGS EP551' 오인식 교정 | **X** | **O** | 커스텀 사전 미지원. **전량 자체개발** (가스 설비 전문용어 사전 + 매칭 로직) | |
| 3. 노이즈 제거 | 머리글/꼬리글/워터마크 제거 | '대외비' 워터마크, 페이지 번호 제거 | **대부분** | **일부** | header/footer를 별도 카테고리로 분류하여 필터 가능 (`exclude=["header","footer"]`). **워터마크 제거는 자체 구현** | |
| 4. 텍스트 정규화 | 특수문자, 인코딩, 공백 통일 | ㎥→m³, ㎏/㎠→kg/cm² 단위 통일 | **부분** | **O** | UTF-8 출력 기본 제공. ㎥→m³ 등 단위 정규화, 특수문자 통일, 공백 처리는 자체 구현 | |
| 5. 메타데이터 추출 | 문서명, 작성일, 버전, 부서 | 사규 개정 이력(버전·시행일) 자동 식별 | **X** | **O** | 구조화된 메타데이터 추출 미지원. header/footer 텍스트에서 regex/NER로 **자체 추출** | |
| 6. 품질 점수 산정 | 글자 수, 구조, 오인식률 종합 | 임계값 미달 시 수동 검수 플래그 | **X** | **O** | confidence 점수 API 미노출. 글자 수/구조 완결성/한국어 비율/오인식 패턴 기반 **자체 산정** | |

### 사전 조사 요약

- **업스테이지 네이티브 커버**: 3단계(노이즈 제거) 대부분
- **부분 지원**: 1단계(구조 태깅), 4단계(인코딩)
- **전량 자체 개발**: 2단계(전문용어), 5단계(메타데이터), 6단계(품질 점수)

**핵심 확인 질문:** 6단계 중 업스테이지가 추가 제공 가능한 기능이 있는지? 특히 confidence 점수 노출 계획과 커스텀 사전 지원 로드맵.

## 참고: API 요청/응답 스펙 (사전 조사)

### 요청 파라미터
```python
response = requests.post(
    "https://api.upstage.ai/v1/document-ai/document-parse",
    headers={"Authorization": f"Bearer {UPSTAGE_API_KEY}"},
    files={"document": open("document.hwp", "rb")},
    data={
        "model": "document-parse",           # 모델명
        "ocr": "auto",                        # "auto" | "force"
        "coordinates": "true",               # 바운딩 박스 좌표
        "output_formats": '["html", "markdown", "text"]',
        "base64_encoding": '["table", "figure", "chart"]',
        "chart_recognition": "true",         # 차트→구조화 데이터
        "mode": "enhanced",                  # "standard" | "enhanced" | "auto"
    }
)
```

### 응답 JSON 구조
```json
{
  "api": "2.0",
  "model": "document-parse-250508",
  "content": {
    "html": "<html>전체 문서 HTML</html>",
    "markdown": "# 제목\n| 열1 | 열2 |\n...",
    "text": "플레인 텍스트"
  },
  "elements": [
    {
      "id": 0,
      "page": 1,
      "category": "table",
      "content": { "html": "<table>...</table>", "markdown": "| ... |", "text": "..." },
      "coordinates": [{"x": 0.1, "y": 0.05}, {"x": 0.9, "y": 0.05}, {"x": 0.9, "y": 0.12}, {"x": 0.1, "y": 0.12}]
    }
  ],
  "usage": { "pages": 3 }
}
```
